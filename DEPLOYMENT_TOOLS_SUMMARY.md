# Deployment Tools Summary

This document provides a summary of all the tools created to help you fully deploy your Document Catalog application on Render.

## Tools Created

### 1. `fix_health_check.py`

**Purpose**: Fixes the SQLAlchemy text() issue in the health check endpoint.

**What it does**:

- Updates the health check in `src/catalog/__init__.py` to use explicit text declaration
- Adds the necessary import: `from sqlalchemy import text`
- Replaces `db.session.execute("SELECT 1")` with `db.session.execute(text("SELECT 1"))`
- Also fixes the same issue in `wsgi.py` and `main_routes.py` if present

**How to use**:

```bash
chmod +x fix_health_check.py
python fix_health_check.py
```

### 2. `add_api_key_auth.py`

**Purpose**: Adds API key authentication to allow programmatic access to API routes without session-based authentication.

**What it does**:

- Generates a secure API key
- Adds a function to check for the API key in request headers or query parameters
- Modifies the `protect_blueprint` function to bypass authentication for API routes when a valid API key is provided
- Outputs the generated API key for you to set as an environment variable

**How to use**:

```bash
chmod +x add_api_key_auth.py
python add_api_key_auth.py
```

### 3. `test_database_connection.py`

**Purpose**: Tests the database connection and diagnoses any issues.

**What it does**:

- Attempts to connect to the database using both SQLAlchemy and psycopg2
- Executes test queries to verify the connection
- Lists all tables in the database
- Provides detailed error information if the connection fails

**How to use**:

```bash
chmod +x test_database_connection.py
python test_database_connection.py
```

## How to Use These Tools

1. **SSH into your Render instance** from the Render dashboard
2. **Upload these scripts** to your Render instance
3. **Run the scripts** in the following order:
   - First, run `fix_health_check.py` to fix the SQLAlchemy issue
   - Then, run `test_database_connection.py` to verify database connectivity
   - Finally, run `add_api_key_auth.py` if you need programmatic API access
4. **Set environment variables** in the Render dashboard:
   - `SITE_PASSWORD`: For web access
   - `API_KEY`: For programmatic API access (use the key generated by `add_api_key_auth.py`)
5. **Restart your application** by exiting the SSH shell or using the "Manual Deploy" button in the Render dashboard
